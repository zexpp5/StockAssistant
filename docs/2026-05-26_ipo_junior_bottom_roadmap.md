# 2026-05-26 · IPO / 次新股触底雷达 roadmap

## 目标

这个模块的真实目标不是做一个普通 IPO 日历，而是：

> 及时发现可能已经接近底部、值得进一步研究的新股 / 次新股，并把明显不能碰的标的提前打回。

页面定位仍然遵守 V2 产品边界：

- `IPO & 次新股` 属于深度研究下的研究观察页，不进入 `AI 推荐`、`AI 组合方案`、`已拉取股票池`。
- 不自动写 `watchlist`，不自动写真实持仓。
- 允许输出 `不碰` / `只观察` / `进入买前审查队列` / `可小仓试探` 这样的研究状态，但它们是 advisory，不是自动交易指令。
- `今日决策台` 最多引用本周事件摘要，不把本页变成今日买入清单。

## 当前事实

截至 2026-05-26：

- A 股次新股触底分是 4 维：`折发行价`、`时间衰减`、`首日溢价`、`较首日跌幅`。
- A 股当前分数本质是“跌幅 + 时间窗口”画像，不包含量价止跌确认。
- 美股次新股来自 NASDAQ priced 24 月窗口 + yfinance 补价；它和 V2 system_universe 是分离的 IPO universe。
- 美股已有流动性 / 从低点反弹等字段，但美股 lock-up 暂未接入。
- 港股 IPO / 解禁 / 次新股暂时只有外链 placeholder。
- 生产可信起点按 2026-05-25 12:10 cutoff 之后观察；在积累 2-4 周样本前，不用绝对分阈值宣称“该看”。

## 设计原则

1. **先防雷，再找机会**  
   红线 gate 优先级高于打分，因为它不改触底公式，但能立刻防止把危险票推到前面。

2. **相对排名先于绝对阈值**  
   样本太薄时，状态分档先用前 10% / 中段 / 末段等相对档。`>=80`、`>=60` 这类绝对阈值至少等 2-4 周样本后再校准。

3. **触底分 != 买入准备度**  
   触底分只回答“像不像跌到底部区域”；买入准备度还要看红线、lock-up、量价修复、基本面和买前审查。

4. **保留 `可小仓试探`**  
   这是最终行动准备度档位之一，不能改名。但它必须放在链路末端，不能由纯跌幅分直接给出。

5. **缺数据要明说，不伪造**  
   美股 lock-up、港股解禁、增发频率等如果没有接入，只能标 `unknown / not_checked`，不能当作已通过。

## 状态链路

推荐使用四档状态：

| 状态 | 含义 | 触发口径 |
|---|---|---|
| `不碰` | 触发硬风险，默认隐藏或置底 | 红线 gate 任一命中 |
| `只观察` | 像底部，但信息不足或确认不足 | 未触发红线，但排名/信号不够强 |
| `进入买前审查队列` | 值得打开个股研究做基本面/行业/技术检查 | 相对排名靠前 + 无红线 + 数据足够 |
| `可小仓试探` | 所有关键检查通过后的最终状态 | 红线通过 + lock-up/解禁安全 + 技术确认 + 买前审查通过 |

注意：`可小仓试探` 只代表系统研究状态，不自动下单、不写持仓。

## Roadmap

### Step 1. 红线 gate 落地与验收

状态：进行中 / 部分已落地。

目标：
- 先把明显不能碰的次新股打回，减少页面噪音。

规则：
- A 股：
  - 30 天内大额解禁：`days_to_unlock <= 30` 且 `pct_of_float >= 30%` 或 `stress_score >= 65`。
  - ST / *ST / 类壳公司名称。
- 美股：
  - SPAC / shell。
  - 现价 `< $1`。
  - 市值 `< $50M`。
  - 30 日日均成交额 `< $200K`。
  - 极端硬剔除可保留：无价、现价 `< $0.3`、市值 `< $10M`。
- 暂不假装检查：
  - 高频增发 / ATM。
  - 美股 lock-up。
  - 港股 lock-up / 解禁。

输出字段：
- `red_lines: [{key, label, detail}]`
- `verdict: "不碰" | null`
- `blocked_by_red_line: true | false`

前端：
- 默认隐藏 `不碰`。
- 提供 `显示不碰` toggle。
- 表格必须能看到每个红线原因。

验收：
- 单测覆盖 A 股解禁红线、ST 红线、美股 SPAC / 仙股 / 微盘 / 低流动性。
- 页面默认不展示 `不碰`，打开 toggle 后展示并能读到原因。
- 不改变 `watchlist` / 真实持仓计数。

### Step 2. 美股 lock-up 数据接入

状态：未做。

为什么重要：
- 没有 lock-up，美股次新池会看不见 90-180 天附近的大额抛压窗口。
- 这是美股次新股“触底 / 继续下跌”判断的核心风险变量。

候选路径：
- 短期：外部链接 + `lockup_status = unknown`，不伪造。
- 中期：接 NASDAQ lock-up / 付费数据源。
- 长期：解析 S-1 / 424B 招股书，提取 lock-up period、到期日、受限股份。

输出字段：
- `lockup_status: "unknown" | "not_applicable" | "safe" | "near_expiry" | "expired"`
- `lockup_expiry_date`
- `days_to_lockup_expiry`
- `lockup_source`

验收：
- 未接入时页面明确显示 `lock-up 未接入`，不能默认为安全。
- 接入后，30 天内到期必须进入红线或高风险提示。

### Step 3. 状态分档

状态：未做。

目标：
- 把纯表格分数翻译成人话状态，减少用户自己脑补。

初版规则：
- `不碰`：任何红线命中。
- `进入买前审查队列`：未触发红线，且在同市场可用候选的前 10%。
- `只观察`：未触发红线，但不在前 10%，或关键数据不足。
- `可小仓试探`：暂不由 Step 3 自动给出，留给 Step 4/5 之后。

为什么用相对档：
- 2026-05-25 12:10 之后才是可信生产 cutoff。
- 样本不足 2-4 周前，不设绝对分数阈值。

输出字段：
- `advisory_status`
- `status_reason`
- `rank_pct`

### Step 4. 触底分 / 买入准备度拆分

状态：未做。

目标：
- 不再让一个 `score` 同时承担“跌得深”和“能不能动手”两件事。

建议字段：
- `bottom_score`：继续沿用当前触底画像。
- `readiness_score`：新增买入准备度。

`readiness_score` 候选维度：
- 红线 gate 是否通过。
- lock-up / 解禁是否安全。
- 量价修复：横盘天数、是否站回 20/50 日均线、低点是否抬高、成交额是否放大。
- 基本面最低可读性：收入增长、毛利率、现金 runway、亏损是否收窄。
- 稀释风险：增发 / ATM / 可转债 / 注册发行。

注意：
- A 股当前没有“有修复迹象”字段，不能在现状说明里写已经做了。
- 技术确认是新增必需项，不是加强项。

### Step 5. 买前审查卡

状态：未做。

目标：
- 把“进入买前审查队列”的标的自动送到买前研究工作台，而不是在 IPO 表里直接喊买。

卡片结构：
- 为什么它像触底。
- 红线是否通过。
- lock-up / 解禁窗口。
- 基本面还缺什么。
- 技术确认是否满足。
- 反对理由 Top 3。
- 下一步：`继续观察` / `进入买前审查队列` / `可小仓试探`。

边界：
- 卡片不写持仓。
- 卡片不自动加入 watchlist。
- 用户手动确认交易后，才进入真实持仓模块。

### Step 6. 早报 / 飞书提醒闭环

状态：未做。

目标：
- 当出现“新进入买前审查队列”的次新股时主动提醒，而不是让用户每天手动翻表。

触发：
- 新进入同市场前 10%。
- 红线从有变无。
- lock-up / 解禁风险解除。
- 技术确认首次通过。

推送文案：
- 只说“进入买前审查队列”或“可小仓试探候选”，不说“买入”。
- 必须附带红线状态和反对理由。
