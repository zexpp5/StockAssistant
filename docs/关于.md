## 1.4 输出口

每天 08:30 自动推**飞书早报卡片**（6 个 section）：

1. 经济日历（持仓 earnings + 政策）
2. 当前建议组合 + 60 天趋势
3. NAV 净值 + Sharpe（"AI 有没有用"）
4. 🎯 次新股触底候选（2026-05-26 接入；🟢 可小仓试探 + 🟡 可研究 跨市场前 3，含 diff 标）
5. 红旗（防御信号升档时显示）
6. 今天做什么

源代码：[morning_brief.py](../stock_research/jobs/morning_brief.py)。中间产物存 `data/latest/*.json`、历史快照存 `data/snapshots/*.json`。

### LLM 输出 citation + 数字 anchor 强约束（**充值后第一件事**）

**为什么必须做**：LLM-in-the-loop 系统会自信地：① 引用错误新闻 ② 混淆同名股（特斯拉 / 特锐德 / 特变电工） ③ 凭空捏造回测数字。

**改动**：[claude_client.py](../stock_research/core/claude_client.py) 加 system prompt 硬约束：

```
1. Citation 强制：每条结论必须附 source URL / 财报段落引用
2. 数字 anchor：关键数字（PE/营收/毛利/PB/ROE）必须从已审计 DuckDB 取，
   不能让 LLM 自己算 — 函数式工具调用（不让 LLM 推理）
3. 失败模式：找不到数据源时输出「数据不可得」，不要编
```

### CONFLICT 历史趋势看板（中等成本）

**为什么需要**：audit_gate 是**事件型**闸门（当日 ≥10% 触发），但缺**趋势**信号 — 数据源缓慢退化（akshare 从 1% → 3% → 5% → 8%）需要趋势看板才能捕捉。
