# 测试用例：真实持仓「持仓纪律 / 触发提醒」

> 状态：评审稿 v1 · 2026-06-03
> 关联需求：[2026-06-03_持仓纪律触发提醒需求文档.md](2026-06-03_持仓纪律触发提醒需求文档.md)

---

## 1. 测试目标

验证「持仓纪律 / 触发提醒」满足以下目标：

- 只服务真实持仓，不创建股票池。
- 只做人工确认后的复查提醒，不自动交易。
- 行情过期时不触发价格型建议。
- 价格区间触发、优先级和文案正确。
- 不污染 AI 推荐、AI 组合、自选股和交易流水。
- 能记录触发历史，用于以后验证规则是否有用。

---

## 2. 测试前置条件

建议使用临时 DuckDB 或测试库。

统一样本：

| 字段 | 值 |
|------|----|
| account | `default` |
| market | `US` |
| symbol | `MRVL` |
| shares | `34` |
| cost_basis_price | `272.54` |
| trade_date | `2026-06-02` |
| plan_type | `event_trade` |
| source_type | `manual_confirmed` |
| validation_status | `unvalidated` |

MRVL 计划：

| 触发类型 | 条件 | 文案 |
|----------|------|------|
| hold_zone | `$285 <= price < $300` | 持有观察，不加仓 |
| take_profit_review | `$300 <= price <= $320` | 止盈复查，可考虑卖 10-15 股锁利润 |
| pullback_review | `$260 <= price <= $265` | 回撤观察，不恐慌，但不加仓 |
| risk_review | `price < $250` | 风险复查，可考虑减半或退出 |
| invalidation_review | `price < $240` | 交易逻辑失败复查，禁止硬扛 |

优先级：

```text
invalidation_review > risk_review > take_profit_review > pullback_review > hold_zone
```

## 3. P0 数据边界用例

### TC-P0-001 只能为真实持仓创建纪律计划

| 项 | 内容 |
|----|------|
| 前置 | `real_holdings` 中不存在 `US:MRVL` |
| 操作 | 尝试创建 MRVL discipline plan |
| 预期 | API 返回 400 或 404 |
| 预期文案 | `真实持仓不存在，不能创建持仓纪律` |
| 负向验证 | `watchlist`、`recommendation_picks`、`portfolio_plans` 均不新增 |

### TC-P0-002 为真实持仓创建 active 计划

| 项 | 内容 |
|----|------|
| 前置 | `real_holdings` 中存在 `US:MRVL` 34 股，成本 `$272.54` |
| 操作 | 创建 1 条 `manual_confirmed` discipline plan |
| 预期 | `real_holding_discipline_plans` 新增 1 条 `status=active` |
| 预期触发 | `real_holding_discipline_triggers` 新增 5 条 |
| 预期标记 | `validation_status=unvalidated`，`auto_trade_allowed=false` |

### TC-P0-003 P0 同一持仓只允许 1 条 active 计划

| 项 | 内容 |
|----|------|
| 前置 | MRVL 已有 active plan |
| 操作 | 再创建一条 active plan |
| 预期 | API 返回冲突，或自动要求先归档旧计划 |
| 预期 | 不允许出现两条同时 active 的 MRVL 计划 |

### TC-P0-004 聊天建议不能自动落库

| 项 | 内容 |
|----|------|
| 前置 | 用户在聊天中收到价格区间建议 |
| 操作 | 不点击页面确认 |
| 预期 | 数据库不新增 discipline plan |
| 关键意义 | 聊天建议不能伪装成已确认真钱交易纪律 |

## 4. P0 触发逻辑用例

### TC-P0-010 行情过期时不触发价格纪律

| 项 | 内容 |
|----|------|
| 前置 | MRVL 最新行情日期为 `2026-06-01`，当前市场日为 `2026-06-03` |
| 操作 | 读取持仓纪律 |
| 预期 | 不触发止盈、减仓、风险复查 |
| 预期文案 | `行情过期，先刷新，暂不触发纪律` |
| 负向验证 | 不写入 `real_holding_discipline_events` 的价格触发事件 |

### TC-P0-011 价格 290 命中持有观察区

| 项 | 内容 |
|----|------|
| 前置 | MRVL 最新有效价格 `$290` |
| 操作 | 计算 discipline trigger |
| 预期触发 | `hold_zone` |
| 预期文案 | `持有观察，不加仓` |
| 预期 severity | `info` |

### TC-P0-012 价格 305 命中止盈复查区

| 项 | 内容 |
|----|------|
| 前置 | MRVL 最新有效价格 `$305` |
| 操作 | 计算 discipline trigger |
| 预期触发 | `take_profit_review` |
| 预期文案 | `止盈复查，可考虑卖 10-15 股锁利润` |
| 预期 severity | `medium` 或 `high` |
| 负向验证 | 不新增 sell trade，不修改 `real_holdings.shares` |

### TC-P0-013 价格 262 命中回撤观察区

| 项 | 内容 |
|----|------|
| 前置 | MRVL 最新有效价格 `$262` |
| 操作 | 计算 discipline trigger |
| 预期触发 | `pullback_review` |
| 预期文案 | `回撤观察，不恐慌，但不加仓` |
| 负向验证 | 不出现 `立即卖出` 文案 |

### TC-P0-014 价格 249 命中风险复查

| 项 | 内容 |
|----|------|
| 前置 | MRVL 最新有效价格 `$249` |
| 操作 | 计算 discipline trigger |
| 预期触发 | `risk_review` |
| 预期文案 | `风险复查，可考虑减半或退出` |
| 预期 priority | 高于普通观察区 |
| 负向验证 | 不自动生成卖出流水 |

### TC-P0-015 价格 239 命中交易逻辑失败复查

| 项 | 内容 |
|----|------|
| 前置 | MRVL 最新有效价格 `$239` |
| 操作 | 计算 discipline trigger |
| 预期触发 | `invalidation_review` |
| 预期文案 | `交易逻辑失败复查，禁止硬扛` |
| 预期 priority | 最高 |
| 负向验证 | 不同时展示较低优先级的 `risk_review` 作为主提醒 |

### TC-P0-016 价格 255 不命中任何明确区间

| 项 | 内容 |
|----|------|
| 前置 | MRVL 最新有效价格 `$255` |
| 操作 | 计算 discipline trigger |
| 预期 | 不触发明确动作，只展示下一触发价 |
| 预期文案 | `下一触发：跌破 $250 风险复查 / 突破 $300 止盈复查` |

### TC-P0-017 区间边界计算

| 项 | 内容 |
|----|------|
| 前置 | 分别测试 `$285`、`$300`、`$320`、`$265`、`$260`、`$250`、`$240` |
| 操作 | 计算 discipline trigger |
| 预期 | 边界归属与需求文档一致 |
| 注意 | `$250` 若规则是 `price < 250`，则 `$250` 本身不触发；若产品改成 `<=250`，测试必须同步更新 |

## 6. P0 审计和副作用用例

### TC-P0-050 触发后写入事件历史

| 项 | 内容 |
|----|------|
| 前置 | MRVL 最新有效价格 `$305` |
| 操作 | 计算并展示触发 |
| 预期 | `real_holding_discipline_events` 新增一条 `take_profit_review` |
| 预期字段 | price、price_trade_date、triggered_at、message 均有值 |

### TC-P0-051 同一价格重复刷新不刷屏

| 项 | 内容 |
|----|------|
| 前置 | MRVL 已在 `$305` 触发过止盈复查 |
| 操作 | 5 分钟内多次刷新页面，价格和 trade_date 不变 |
| 预期 | 不重复写入多条完全相同的触发事件 |
| 允许 | 更新 last_seen_at 或展示同一 active alert |

### TC-P0-052 用户确认触发提醒

| 项 | 内容 |
|----|------|
| 前置 | 存在未确认的触发事件 |
| 操作 | 用户点击 `已查看` |
| 预期 | `user_ack_status=acknowledged`，`user_ack_at` 有值 |
| 负向验证 | 不修改交易流水和持仓数量 |

### TC-P0-053 暂停计划后不再触发

| 项 | 内容 |
|----|------|
| 前置 | MRVL active plan |
| 操作 | 用户暂停计划 |
| 预期 | `status=paused` |
| 操作 | 价格进入 `$300-$320` |
| 预期 | 不生成触发事件，只显示 `计划已暂停` |

### TC-P0-054 归档计划保留历史

| 项 | 内容 |
|----|------|
| 前置 | MRVL active plan 且已有触发历史 |
| 操作 | 用户归档计划 |
| 预期 | `status=archived` |
| 预期 | 触发历史仍可查询，不被删除 |

### TC-P0-055 不污染 AI 推荐和组合

| 项 | 内容 |
|----|------|
| 前置 | 创建 MRVL discipline plan |

| 预期 | 不因创建 discipline plan 增加、删除或改分 |
| 关键意义 | 持仓纪律不能变成 AI 推荐入口 |

### TC-P0-056 不污染 watchlist

| 项 | 内容 |

| 前置 | MRVL 不在手动自选股 |
| 操作 | 创建 MRVL discipline plan |
| 预期 | `watchlist` 不新增 MRVL |
| 关键意义 | 真实持仓纪律不等于自选股配置 |

### TC-P0-057 不自动生成卖出流水

## 8. 验收查询示例

创建计划后，不应改变以下事实表：

```sql
SELECT COUNT(*) FROM recommendation_picks;
SELECT COUNT(*) FROM portfolio_plans;
SELECT COUNT(*) FROM watchlist;
SELECT SUM(shares) FROM real_holdings WHERE market='US' AND symbol='MRVL';
```

触发后必须能查询事件：

```sql
SELECT symbol, price, price_trade_date, message, user_ack_status
FROM real_holding_discipline_events
WHERE market='US' AND symbol='MRVL'
ORDER BY triggered_at DESC;
```

## 9. 最小通过标准

P0 通过必须同时满足：

- MRVL 有真实持仓时才能创建纪律计划。
- 页面能显示计划类型、规则可信度和下一触发价。
- stale 行情只提示刷新，不触发卖出或减仓复查。
- 价格进入区间时触发正确文案。
- 触发后不自动交易。
- 所有触发事件可追溯。
- AI 推荐、AI 组合、自选股、真实交易流水不被污染。
