## 三、不需充值但 backlog（DSA 对比后新增）

### 3.1 数据新鲜度单股告警（低成本，下次 daily_audit 改造时顺手加）

**现状**：audit_gate 有 max_age 全局守卫（36h），但**单股层面**没有"这只股票 X 天没刷新"告警。

**改动**：
- `audit.py audit_stock()` 返回值加 `freshness_age_hours`
- `daily_audit.py` 控制台高亮 `freshness > 72h` 的标的
- 加入 morning_brief section 4（红旗）

### 3.2 LLM 输出 citation + 数字 anchor 强约束（B 路线充值前提，**充值后第一件事**）

**为什么必须做**：DSA 反面教材教训 — LLM-in-the-loop 系统会自信地：
- 引用错误新闻
- 混淆同名股（特斯拉 / 特锐德 / 特变电工）
- 凭空捏造回测数字

**改动**：[stock_research/core/claude_client.py](../stock_research/core/claude_client.py) 加 system prompt 硬约束：

```
1. Citation 强制：每条结论必须附 source URL / 财报段落引用
2. 数字 anchor：关键数字（PE/营收/毛利/PB/ROE）必须从已审计 DuckDB 取，
   不能让 LLM 自己算 — 函数式工具调用（不让 LLM 推理）
3. 失败模式：找不到数据源时输出「数据不可得」，不要编
```

### 3.3 CONFLICT 历史趋势看板（中等成本）

**为什么需要**：audit_gate 是**事件型**闸门（当日 ≥10% 触发），但缺**趋势**信号 — 数据源缓慢退化（如 akshare 从 1% CONFLICT 逐周升到 3% → 5% → 8%）需要趋势看板才能捕捉。

**改动**：
- morning_brief section 1（regime gate）下方加 "audit 健康度" 一行
- 显示过去 7 天 CONFLICT 比例 sparkline
- > 5% 持续 3 天以上 = 数据源在退化的早期信号

baostock 已经在跑，dashboard 没暴露这层信息。可选改动：watchlist 编辑 tab 旁加 "数据可信度" 列，显示 akshare/baostock 是否一致。**不紧急**，audit snapshot 已经在写。

## 七、给一年后的你（v2 的核心 lessons）

1. **多源 ≠ 多源验证**：DSA 对比分析的最大收获 — 加数据源时必须同时设计 reconcile 逻辑，不然就是"看起来很丰富的不可信系统"
2. **付费源接入前先想免费替代**：本轮 baostock 就是省下 Tushare ¥200/年的零成本替代品；下次也先 grep 开源圈再考虑充值
3. **闸门 > 报告**：audit 早已在产 HIGH/MEDIUM/LOW/CONFLICT 标签，但**没人消费这个信号**等于白做；改造为生产路径硬闸门才有用
4. **LLM 不在关键路径**：你的护城河（factor 模型 + 13F + 跨源审计）方向是对的；B 路线充值后 LLM 仅作辅助，且必须带 citation + 数字 anchor 硬约束
