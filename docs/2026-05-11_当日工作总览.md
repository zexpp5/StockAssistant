## 📌 主线 4：OpenBB → yfinance + 意外挖出真实 HIGH 信号

### 起因

defense_watcher 每次跑都 `WARNING: OpenBB options chains failed: No module named 'openbb'`，用户问"OpenBB 缺失警告是什么意思"。

解释：OpenBB 是免费金融数据平台，系统只在 [`stock_research/core/options_signals.py`](../stock_research/core/options_signals.py) 用它拉 SPY 期权链算 Put/Call Ratio（PCR，Pan & Poteshman 2006 的 leading indicator）。

### 决策

用户要求"系统准确，不舍弃必要的"。但发现：
- OpenBB 装到系统 python (Homebrew 3.14) 触发 PEP 668
- 用 `--break-system-packages` dry-run 显示会**降级**：fastapi 0.136→0.128 / starlette 1.0→0.52 / uvicorn 0.46→0.40，而 `api/main.py` 真用 fastapi
- **关键发现**：OpenBB 在这里其实就是一层包装，源码 `obb.derivatives.options.chains(symbol, provider="yfinance")` 最终还是调 yfinance

→ 直接重写为 yfinance 原生 API，去掉一层间接，避免 600MB 依赖 + 不降级 fastapi。

### 结果（最大收益）

修好后第一次跑出：
```
PCR (volume) = 1.274 → BEARISH（CBOE 阈值 ≥ 1.2）
severity: NONE → HIGH
```

**这是个真实信号，之前因为 OpenBB import 失败被 silent skip 一直显示 NONE。**用户说"要系统准确"的直接收益。

morning_brief 自动联动：
- Section 1 橙色 + "减仓 30-50%，停止买入"
- Section 5 "regime = HIGH：今天只减仓不加仓"
- 红旗 section 出现 `regime = HIGH (见上方)`

### Sanity check

| 字段 | 值 | 评估 |
|---|---|---|
| PCR(volume) | 1.274 | ⚠️ 触阈值但不极端（1.5+ 才极端） |
| PCR(OI) | 3.37 | 偏高但学术界仅用 volume PCR（OI 含 deep-OTM puts 干扰） |
| 触发分支 | check_market_regime → signal_volume.severity = HIGH | 仅 PCR；VIX / 200MA 未升档 |

**判断**：PCR leading VIX 1-3 天，可能是这种早预警。盯紧 VIX/200MA 是否跟着升档；若 PCR 回落 < 1.2 watcher 会"降档静默"。

## 📌 主线 5：飞书渲染 3 轮迭代

最严重的实测发现：**Unicode 8 级 block (`▁▂▃▄▅▆▇█`) 在飞书 markdown 客户端字体下被全部渲染成同高度实心块**。

### v1（unicode block，失败）

```
• WDC 15.0% · F-Score 6 · 综合 +1.11 · ████████ +69.0% 60d
```
用户截图：全黑矩形，看不出趋势。原因：飞书 markdown 字体不支持 unicode block 渐变高度。

### v2（emoji 5 档，区分度低）

降为 5 档 `🚀 / 📈 / ➡️ / 📉 / 💀`，但 12 只里 9 只都 ≥+20% 触发 🚀，几乎一致。

### v3（emoji 7 档 + 图例，最终版）

7 档 + 阈值更细：
- 🚀 ≥+50% 飙涨
- 📈 +15~50% 强涨
- ↗️ +3~15% 小涨
- ➡️ ±3% 横盘
- ↘️ -3~-15% 小跌
- 📉 -15~-50% 强跌
- 💀 ≤-50% 暴跌

**关键补救**（按 memory 偏好"飞书 card 要自带解释"）：
- markdown 端 section_picks 末尾加 `TREND_LEGEND_SHORT` 行
- card 端 section 2 美股清单下方加 note 元素（灰色小字，同灯色对照风格）

实测分布：🚀×3 / 📈×6 / ↗️×2 / ➡️×1 — 区分度合理。

### 经验

- **Unicode block sparkline 在飞书不可用**（PC 客户端字体限制）
- emoji 跨平台兼容性最稳，但要给图例（新人看不懂"🚀 vs 📈"）
- 函数名保留 `_sparkline` 内部换实现，caller 无感切换（向后兼容）
