## 二、约束器执行优先级（美股 · [optimize_portfolio.py](../stock_research/jobs/optimize_portfolio.py)）

**铁律：从个股层 → 组合层 → 流动性/成本层**，越靠后的约束**只能下调**前面的权重，不能上调。

| 顺序 | 约束器 | 作用层 | 位置 |
|---|---|---|---|
| 1 | Markowitz / risk_aware_optimize | 组合优化 | [optimize_portfolio.py:265-337](../stock_research/jobs/optimize_portfolio.py#L265) |
| 2 | **半 Kelly 单股上限** (kelly_cap, default 0.5) | 个股层 cap | [optimize_portfolio.py:343-360](../stock_research/jobs/optimize_portfolio.py#L343)  ← 2026-05-12 接入 |
| 3 | 行业敞口 ≤ 25% (cap_by_industry) | 行业层 cap | [portfolio_constraints.py:35](../stock_research/core/portfolio_constraints.py#L35) |
| 4 | ADV 限流 ≤ 5% (cap_by_adv) | 流动性 cap | [portfolio_constraints.py:163](../stock_research/core/portfolio_constraints.py#L163) |
| 5 | 交易成本扣减 (apply_transaction_cost) | 成本汇报 | [portfolio_constraints.py:231](../stock_research/core/portfolio_constraints.py#L231) |

**为什么这个顺序：**
- 半 Kelly 必须**先做**：先按"个股暴雷保护"压住单股，再算行业总和；反过来行业 cap 后再压 Kelly 会让 cash 飙升。
- ADV 在行业 cap **之后**：行业 cap 是组合层意愿，ADV 是市场层物理约束；先满足意愿再被市场打折。
- 成本是事后汇报，不改权重（只算 net alpha）。

## 四、风险闸门（独立于仓位优化器，全局 override）

| 闸门 | 触发条件 | 行为 | 位置 |
|---|---|---|---|
| VIX 抬升 | VIX > 阈值 | 降仓 / regime 升级 | [regime_filter.py](../stock_research/core/regime_filter.py) |
| 200MA 跌破 | SPY < SPY.MA200 | Faber 趋势切换 | [regime_filter.py:71](../stock_research/core/regime_filter.py#L71) |
| 单股 -X% 止损 | ATR-proxy 动态 stop_pct（7%-25% 区间）破线 | morning_brief 标红清仓 | [morning_brief.py:section_holdings_stoploss](../stock_research/jobs/morning_brief.py)  ← 2026-05-12 接入 |
| PCR 极端 | Put/Call > 1.5 | 警示 | [options_signals.py](../stock_research/core/options_signals.py) |
| 宏观恶化 | 多 macro 指标合议 | 风控降仓 | realtime_defense |
| 跨源 CONFLICT | conflict 比例 ≥ 10% | 强制 dry-run | [audit_gate.py](../stock_research/core/audit_gate.py) |
| 因子 IC 失效 | IC<0.03 或 |IR|<0.30 | 强制 dry-run | [factor_ic_gate.py](../stock_research/core/factor_ic_gate.py) |

**冲突解决规则：**
1. 跨源 CONFLICT 闸门 + 因子 IC 闸门 **最高优先**：触发即 dry-run，不出选股。
2. regime（VIX/200MA/宏观）→ 调整 gross_exposure_cap，影响仓位优化的 cash_pct 参数。
3. 个股止损 ≠ 仓位优化器输入：只影响 morning_brief 持仓警告，**不自动平仓**（让用户决定）。

## 五、新增约束器接入清单（避免再写死代码）

新加约束器函数时，必须按本清单走完才算"接入"：

- [ ] 函数定义在 [portfolio_constraints.py](../stock_research/core/portfolio_constraints.py) 或专门模块
- [ ] 论文 / 学术依据写在 docstring
- [ ] 单元测试或 inline `python3 -c` 自测
- [ ] **在主路径调用方加入调用代码**（不只是 import）
  - 美股仓位：optimize_portfolio.py
  - A 股仓位：apply_a_share_constraints.py
  - 选股软红旗：daily_picks_v5.py
- [ ] 函数参数加入 result["constraints"] 输出（让结果可审计）
- [ ] CLI 加 `--xxx` 开关（可关闭，便于消融测试）
- [ ] 本文档"约束器执行优先级"表加一行
- [ ] grep 验证 `调用方文件名 → 函数名` 命中

**反例（不要复刻）：**
- 🔴 kelly_cap 函数 2026-04-XX 写好，到 2026-05-12 才发现全仓库 0 调用 → 整 8 个月空转
- 🔴 apply_stop_loss 函数只在 stress_test.py 离线用，生产 morning_brief 从未调用
- 🔴 a_share_picks / build_plan_a_v5 完全绕过 portfolio_constraints 模块

## 六、消融测试要求（新接入功能必做）

任何"修改主路径行为"的接入，必须有对照消融：

| 接入项 | 消融对照 |
|---|---|
| 新约束器（如 kelly_cap） | `--kelly-fraction 0` 关掉 vs 默认 0.5 vs 1.0，比较 Sharpe / 回撤 / 换手 |
| 新止损（如 ATR-proxy） | 与固定 -15% 对照，比较 walk_forward 12 月期内的"被止损出局后错过反弹"次数 |
| 新软红旗（如 Z/M-Score） | recommended pool 加红旗前后的命中率（discovery_tracking 表对比） |

走 [walk_forward_backtest.py](../stock_research/jobs/walk_forward_backtest.py)（每周一自动跑）做长期 OOS 校验。
