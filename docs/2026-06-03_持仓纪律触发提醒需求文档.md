# 需求文档：真实持仓「持仓纪律 / 触发提醒」

> 状态：评审稿 v1 · 2026-06-03
> 关联：我的池子 → 我的持仓（`real_holdings` / 今日持仓体检 / 交易计划复盘）

---

## 0. 一句话需求

> 「我真实买入一只股票后，需要把买前或买后的持仓纪律记录在系统里。系统只在价格触发时提醒我复查，不自动卖出、不自动改仓。」

这不是 AI 推荐，也不是自动交易。它是用户真实持仓的风险纪律层，用来回答：

- 这笔交易原来为什么买？
- 什么价格区间只观察？
- 什么价格触发止盈复查？
- 什么价格触发风险复查？
- 什么价格说明交易逻辑失败？
- 触发时系统应该提醒什么，而不是替用户直接交易。

---

## 1. 产品定位和边界

本功能属于 **我的池子 → 我的持仓**，只服务 `real_holdings` 里的用户真实仓位。

硬边界：

- 触发提醒只读真实持仓和行情，不创建股票池。
- 不写 `recommendation_picks`，不改 AI 推荐排名。
- 不写 `portfolio_plans`，不改 AI 组合目标仓位。
- 不自动写 `watchlist`。
- 不自动生成买入、卖出、加仓或减仓流水。
- 所有“卖出 / 减半 / 锁利润”文案必须是“复查提醒”，不能是“立即执行”。
- 行情过期时，不能触发价格型建议，只能提示“先刷新行情”。

本功能允许和今日持仓体检同屏展示，但语义不同：

| 模块 | 作用 | 是否可信为交易建议 |
|------|------|--------------------|
| 今日持仓体检 | 系统通用 guardrail，如浮亏、集中度、数据缺失 | 只做风险提示 |
| 持仓纪律 / 触发提醒 | 用户确认过的单票交易计划 | 只做触发复查，不自动交易 |
| AI 推荐 / AI 组合 | 模型筛选和目标组合 | 不等于真实持仓纪律 |

---

## 2. 为什么要做

用户已经拿真钱买入后，最危险的不是“没有观点”，而是观点漂移：

| 现状 | 风险 |
|------|------|
| 交易计划只在聊天里 | 过几天忘记，无法复盘 |
| 系统只有通用浮亏阈值 | 无法区分核心仓、事件交易、波段交易 |
| 触发价没有记录 | 价格剧烈波动时临时决策，容易追涨杀跌 |
| 行情滞后时仍显示盈亏 | 可能误触发错误风控 |
| AI 建议和真实持仓混在一起 | 用户可能误以为系统已经确认卖出或加仓 |

本功能要把纪律写成可检查、可追溯、可复盘的结构化规则。

## 3. MRVL 示例计划

用户真实持仓：

| 字段 | 值 |
|------|----|
| 股票 | `MRVL` |
| 市场 | US |
| 数量 | 34 股 |
| 成本价 | `$272.54` |
| 成本金额 | `$9,266.36` |
| 计划类型 | 事件驱动高弹性仓 |

人工确认计划示例：

| 价格区间 | 触发提醒 |
|----------|----------|
| `$285-$300` | 持有观察，不加仓 |
| `$300-$320` | 止盈复查，可考虑卖 10-15 股锁利润 |
| `$260-$265` | 回撤观察，不恐慌，但不加仓 |
| 跌破 `$250` | 风险复查，可考虑减半或退出 |
| 跌破 `$240` | 交易逻辑失败复查，禁止硬扛 |

注意：以上不是系统自动卖出指令。它是人工风控计划，必须在页面标注 `人工确认计划` 和 `未回测纪律`。

---

## 4. 这条规则的计算逻辑

### 4.1 输入

MRVL 这组规则来自四类输入：

1. **真实成本基准**  
   成本价 `$272.54`，数量 34 股，总成本约 `$9,266.36`。

2. **交易性质**  
   MRVL 不是低位核心仓，而是事件驱动高弹性仓。触发原因来自 AI / ASIC / NVIDIA 合作 / Computex 热度。

3. **系统因子状态**  
   系统最新判断显示 MRVL 在系统池内，但没有进入美股 Top20。此前重算时 MRVL 美股内部约第 30，主要风险是 1 年涨幅过热、近 1 月涨幅过大、反转分为 0。

4. **风险阈值习惯**  
   真实持仓体检已有通用 `loss_review_pct = -12%`。单票事件交易可以更早设置风险复查线，避免等到系统通用浮亏阈值才处理。

### 4.2 阈值解释

按成本价 `$272.54` 计算：

| 阈值 | 相对成本 | 逻辑 |
|------|----------|------|
| `$300` | 约 `+10.1%` | 事件交易达到双位数收益，进入止盈复查 |
| `$320` | 约 `+17.2%` | 快速上冲后容易情绪拥挤，优先锁一部分利润 |
| `$265` | 约 `-2.8%` | 普通回撤，不足以说明交易失败 |
| `$260` | 约 `-4.6%` | 仍属观察区，但禁止继续加仓摊成本 |
| `$250` | 约 `-8.3%` | 事件溢价明显回吐，进入风险复查 |
| `$240` | 约 `-11.8%` | 接近系统通用 `-12%` 风险线，视为交易逻辑失败复查 |

所以这不是拍脑袋的“魔法价格”。它是用成本价、仓位性质、事件热度、系统过热标记和通用风险阈值拼出来的风控纪律。

### 4.3 可信度

可信度必须分层：

| 维度 | 结论 |
|------|------|
| 作为预测模型 | 不可信，不能预测未来涨跌 |
| 作为纪律提醒 | 有用，可以减少临场情绪决策 |
| 作为自动卖出信号 | 不允许，必须人工复查 |
| 是否已回测 | 未回测，默认标记为 `manual_guardrail_unvalidated` |
| 是否可直接推广到别的股票 | 不可以，必须按持仓类型和成本重算 |

产品默认显示：

```text
规则可信度：人工风控纪律，未回测；触发后仅提醒复查，不自动交易。
```

## 5. 功能需求

### 5.1 持仓纪律计划

每条当前持仓可以绑定 0 到多条纪律计划。P0 只允许 1 条 active 计划。

计划字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| plan_id | 是 | 计划 ID |
| holding_id | 是 | 绑定真实持仓 |
| account | 是 | 账户 |
| market | 是 | 市场 |
| symbol | 是 | 股票代码 |
| plan_type | 是 | core / event_trade / swing / risk_only |
| source_type | 是 | manual_confirmed / ai_draft / imported_note |
| validation_status | 是 | unvalidated / paper_tracked / backtested |
| status | 是 | active / paused / archived |
| cost_basis_price | 是 | 创建计划时的成本价 |
| shares_snapshot | 是 | 创建计划时的持股数 |
| thesis | 否 | 买入逻辑 |
| invalidation_note | 否 | 什么情况说明逻辑失败 |
| created_at | 是 | 创建时间 |
| confirmed_at | 否 | 用户确认时间 |
| updated_at | 是 | 更新时间 |

### 5.2 触发条件

每个计划有多条触发条件。

触发字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| trigger_id | 是 | 触发条件 ID |
| plan_id | 是 | 所属计划 |
| trigger_type | 是 | hold_zone / take_profit_review / pullback_review / risk_review / invalidation_review |
| comparator | 是 | between / gte / lte / lt |
| price_min | 条件必填 | 区间下限 |
| price_max | 条件必填 | 区间上限 |
| severity | 是 | info / medium / high / critical |
| action_label | 是 | 展示文案 |
| suggested_size_text | 否 | 例如“可考虑卖 10-15 股” |
| auto_trade_allowed | 是 | P0 固定 false |
| priority | 是 | 多个条件同时命中时的优先级 |
| rationale | 是 | 为什么设置这条触发 |

### 5.3 页面展示

在「我的持仓」表格新增一列：

```text
持仓纪律
```

建议展示结构：

```text
MRVL
事件交易 · 人工确认 · 未回测
下一触发：跌破 $250 风险复查 / 突破 $300 止盈复查
数据：最新行情 2026-06-03，若过期先刷新
```

触发态示例：

| 当前状态 | 展示 |
|----------|------|
| 行情过期 | `行情过期，先刷新，暂不触发纪律` |
| 未触发 | `持有观察，不加仓` |
| 触发止盈 | `止盈复查：价格进入 $300-$320，可考虑卖 10-15 股` |
| 触发风险 | `风险复查：跌破 $250，检查是否减半或退出` |
| 触发失效 | `交易逻辑失败复查：跌破 $240，禁止硬扛` |

### 5.4 行情新鲜度

价格触发必须先检查行情时间。

P0 规则：

- 美股：最新交易日距离当前交易日超过 1 个交易日，视为 stale。
- 港股 / A 股：同理按市场交易日判断。
- 行情 stale 时不输出“卖出 / 减仓 / 止盈”类触发，只显示“先刷新行情”。

### 5.5 审计和复盘

每次触发都应记录：

- 触发时间。
- 当时价格。
- 触发条件。
- 展示给用户的文案。
- 用户是否确认、忽略、暂停计划、修改计划。

这部分用于后续回答“规则到底准不准”。

## 7. 反误导文案规则

禁止文案：

```text
立即卖出
必须减半
系统建议清仓
保证止盈
```

允许文案：

```text
触发风险复查
触发止盈复查
可考虑分批锁定利润
行情过期，先刷新
该纪律未回测，仅作人工复查提醒
```

---

## 8. P0 / P1 范围

P0：

- 支持为真实持仓创建 1 条 active 人工纪律计划。
- 支持价格区间和阈值触发。
- 持仓页展示持仓纪律、下一触发价、数据状态。
- 行情过期时阻断价格触发。
- 触发后只提醒，不自动交易。
- 记录触发历史和用户确认状态。

P1：

- 支持多计划版本和历史归档。
- 支持按成本自动生成建议阈值草稿。
- 支持把触发后的实际处理结果纳入复盘。
- 支持回测类似纪律：触发后 1d / 5d / 20d 的收益和回撤。
- 支持不同持仓类型的模板：核心仓、事件交易、波段交易、ETF 风控。

## 9. 未解决问题

- 是否允许没有真实持仓时先创建“预案”？P0 不支持，避免混成 watchlist 或买前研究。
- 是否支持期权和 ETF 的纪律模板？P0 只支持股票价格触发。
- 是否自动从聊天记录生成规则？P0 不自动。必须用户在页面确认后才落库。
- 是否允许自动计算止损价？P0 只允许生成草稿，不允许直接 active。
