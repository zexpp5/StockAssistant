### ~~P0 原计划~~（保留为参考）

**问题**：[build_stock_dashboard_html.py::compute_plan_forward_track()](../build_stock_dashboard_html.py#L3718)

```python
# 当前实现：
plan_list = plan.get("plan_v5") or plan.get("plan_v6") or plan.get("plan") or []
inception_date = _find_plan_inception_date()  # MIN(ts) from DuckDB → 永久不变
```

`plan` 是函数入参，传入的是**今天最新** `plan_a_v5.json` 的内容；`inception_date` 永久冻结在第一次写库的日期（5-09）。

**带来的 bug**：如果明天 v6 选股把 MU 换成 AAPL，AI 方案模拟会**把 AAPL 从 5-08 起的真实历史轨迹算进 NAV**，仿佛"你 5-08 就买了 AAPL"——这是 look-ahead bias，不可实操。

**修法（P0 最小改动）**：把 tickers 也冻结到 inception 那天的 plan 快照。

```python
# 修后：
inception_plan = _load_inception_plan_from_duckdb()  # 读 DuckDB 里 inception 那天的 plan
plan_list = inception_plan.get("plan_v5") or ... or []
inception_date = _find_plan_inception_date()
```

**结果**：从 buy-and-hold 视角忠实跟踪"5-08 那天锁定的 12 只"，没有 look-ahead bias。但**没有动态调仓的价值**（A 类设计）。
