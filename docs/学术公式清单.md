# StockAssistant 学术公式清单

**目的**：盘点系统当前接入的全部学术公式 / 风控模型，按**精确状态**分类。

**维护规则**：每次新增 / 拒绝 / 升级一个学术公式，更新本文档对应行。任何指标
进入**生产主路径**必须通过 [docs/系统模块优先级.md](系统模块优先级.md#七指标因子准入门槛)
的**门 3 消融测试**。

**状态枚举（7 档精细化，回应四审"分类太粗"批评）**：
- 🟢 **生产主路径** — 选股 / 仓位优化 / 风控决策直接采纳，影响实盘
- 🔵 **监控展示** — 在 morning_brief / dashboard 显示，但不进决策（用户参考）
- 🟡 **函数库待消融** — 代码已写，未接主路径，待 walk-forward 验证
- ✅ **消融通过** — 经历完整门 3 消融批准，已或可立即接入
- ❌ **消融拒绝** — 走过门 3 被拒，明确不上生产（留模块作历史 / 未来 universe 切换时复用）
- ⏸ **数据源阻塞** — 代码就绪，等数据 / API key / 重抓
- 🚫 **明确不做** — 学术校正失效或与已有指标冗余，浪费工程

## 九、维护规则

1. **新增公式必须经过**[docs/系统模块优先级.md](系统模块优先级.md) **门 3 消融**（4 件套：Sharpe Δ≥0.3 / MDD-2pp / turnover≤50% / 错杀率≤15%）。
2. **每次门 3 测试** → 落地 [docs/ablation_walk_forward_*.md](.)。
3. **消融拒绝的指标** → 留模块标 ❌，记录原因。
4. **数据 / API 阻塞** → 留 ⏸ 状态。
5. **不补清单** → 不浪费时间复活。
6. **状态升级路径**：⏸ → 🟡 → ✅ → 🟢 / 🔵；或 ❌（拒绝）。
7. **每次自评** 必须老实 — "函数存在 ≠ 生产闭环完成"。
