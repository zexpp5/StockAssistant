# 测试用例：真实持仓「卖出 / 平仓」记录功能

> 状态：评审稿 v2 · 2026-06-02
> 关联需求：[2026-06-02_卖出记录需求文档.md](2026-06-02_卖出记录需求文档.md)

---

## 1. 测试目标

验证真实持仓从“买入 lot 录入”升级为“交易流水事实源 + 当前持仓聚合缓存”后，能正确支持：

- 买入、加仓、部分卖出、全部卖出。
- 加权平均成本。
- 已实现盈亏、当前浮动盈亏、累计收益。
- 多币种汇率和手续费折算。
- 老数据迁移。
- `fetch_all_real_holdings()` 兼容字段和下游读者回归。
- rebuild 排序回放、同日交易顺序、清仓轮次。
- 持仓体检、行情刷新、收益曲线只读取当前未清仓持仓。
- 产品边界：卖出是用户事后记账，不是 AI 建议，不自动改自选股。

---

## 2. 测试前置条件

建议用临时 DuckDB 或测试库执行，避免污染生产库。

统一测试参数：

| 参数 | 值 |
|------|----|
| account | `default` |
| USD/CNY | `7.0` |
| HKD/CNY | `0.9` |
| CNY/CNY | `1.0` |
| 手续费默认币种 | 同股票交易币种；如测试用例特别说明则按说明 |
| 日期基准 | 当前日期不早于 `2026-06-02` |
| 幂等键 | API 写入请求必须支持 `client_request_id` / `idempotency_key` |

统一股票样本：

| 市场 | 股票 | 币种 |
|------|------|------|
| US | `MCD` | USD |
| HK | `9992.HK` | HKD |
| CN | `600519.SS` | CNY |

说明：

- 用例前置以每条 TC 的“前置”字段为准。
- 明确写“已完成 TC-xxx”的用例按序列接续；没有写接续关系的用例应视为独立建仓场景。
- 所有买入手续费默认计入成本基；所有卖出手续费默认从已实现盈亏中扣减。
- 原币均价用原币成交价和股数加权，不由 RMB 成本除当前汇率倒推。
- RMB 成本按每笔交易锁定汇率和手续费计算；未实现盈亏按当前价格和当前汇率盯市。

## 4. API 用例

### TC-API-001 POST `/api/real-holdings`

| 项 | 内容 |
|----|------|
| 操作 | 提交首次买入 payload |
| 预期 | 返回 `status=ok`，包含当前持仓 id，创建 buy trade |

## 12. 通过标准

P0 通过标准：

- 所有后端账本用例通过。
- 迁移脚本可重复执行且不重复记账。
- 多币种、手续费、超卖拦截与手算一致。
- 亏损卖出、部分卖出后再加仓、乱序补录、同日排序都与手算一致。
- 清仓后再买按新 `position_epoch` 计算，不污染新一轮成本。
- 撤销最近一笔 trade 后，当前持仓和收益摘要能正确 rebuild。
- 当前持仓可从交易流水重建。
