## 3. 方案设计：US 严筛试运行 overlay

### 3.1 严筛口径（三道筛）

在 **现有 US 推荐结果之上**做一层只读过滤，不重算分数：

| 筛 | 规则 | 依据 | 状态标注 |
|---|---|---|---|
| 筛1 · 取头部 | 只保留 **US buy 内 market_rank 1-5**（见下方定义） | Top1-5 实测 alpha +0.22% / 胜率 52.3%，Top6-20 拖后腿 | 较稳 |
| 筛2 · 避动量陷阱 | **momentum < 80** | 高动量组 alpha −2.47%，低动量组 +1.57%，当前反转市动量反向 | 🧪 **实验** |
| 筛3 · 硬拦过热 | 剔除 **OVERHEATED_1Y** | 该红旗 13 样本胜率 0% / alpha −4.69% | 较稳 |
| 产出 | 每天最多 **3-5 只** "US 严筛候选" | — | — |

实测分层证据（同一批 US 样本）：

| 口径 | 样本 | 胜率 | 1D alpha |
|---|---|---|---|
| 全部 US buy | 260 | 38.5% | −0.64% |
| US Top 1-5 | 65 | 52.3% | +0.22% |
| **US Top 1-5 且 momentum<80** | **49** | **59.2%** | **+1.59%** |

> **重要：上表是历史成熟样本的分层证据，不是今日 3-5 只候选已经验证通过。今日候选的 outcome 需要等未来交易日回填。**

> ✅ **「Top 1-5」定义已钉死**：取最新 `recommendation_picks` 中 `market='US'` 且 `signal='buy'` 的行，按 `total_score DESC NULLS LAST, symbol ASC` 在 US 市场内重排，得到 `market_rank`；严筛只保留 `market_rank <= 5`。
> 注意：这不是数据库里的全局 `rank <= 5`。例如今天 SQM 的全局 `rank=41`，但在 US buy 子口径内是 `market_rank=1`，因此可以进入严筛。

### 3.2 只读 overlay 架构（不改公式）

- 新增一个**独立只读 job / 视图**，当前候选只读最新 `recommendation_picks`（US buy 子口径）+ `factor_scores_json`（momentum）+ `risk_flags_json`（OVERHEATED_1Y）。
- 历史证据摘要单独读 `strategy_failure_diagnosis.json` / `pick_outcomes`，只用于解释"为什么这个窄口径比 Top20 更好"。
- 输出一个独立产物（如 `data/latest/us_strict_trial.json`），**不回写** recommendation_picks / watchlist / real_holdings / 任何公式。
- dashboard 在 AI 推荐页**顶部单独开一个区**"🧪 US 严筛试运行"，与正式主榜分开渲染。

字段可得性已核对 ✅：
- 当前展示字段：`symbol` / `name` / `rank`（全局审计用）/ `total_score` / `factor_scores_json.momentum` / `risk_flags_json` / `entry_price`。
- 历史证据字段：`pick_outcomes.alpha_pct` / `is_success` / `horizon`。
注意：最新批次的 `pick_outcomes` 尚未成熟，不能把历史 alpha 展示成"今天这 5 只已经验证通过"。

### 3.3 安全边界（写死在产品里）
- 必须进**买前研究**才看详情，不自动买、不自动写持仓。
- 文案 advisory 不 directive：「严筛候选 · 仅供买前研究」，禁止「建议买入」。
- **自动降级 / 退回**：严筛口径 1D alpha 连续转负先降级为折叠研究观察；累计样本足够后仍为负才撤下该区（阈值见 §7.3）。

## 4. 两道保险（本会话增量，防过拟合）

> 严筛方案方向正确，但有两个坑，写进产品避免误导：

**保险1 · momentum<80 是第一版筛选硬阈值，但必须标"🧪实验口径"。**
该阈值是在 **49 个样本**上挑出来的（59.2% 胜率），样本小、统计不稳，本质是"专门躲这波回调"。**一旦市场从反转市切回趋势市，这条会反过来亏**（典型 data snooping）。→ 第一版筛选逻辑里它确实是硬过滤；"🧪实验"的含义是页面必须标注、回溯必须单独盯、证据转弱必须退回，不能把它包装成长期已验证规则。

**保险2 · 严筛名单标"博反弹候选 · 非稳健票"。**
今天严筛留下的 **NXPI 恰在过去 5 天"最差 12 只"里**（alpha −5.5%）。它能入选正因为"跌多了→动量低→被当超跌反弹"。逻辑自洽（抄底），但等于**赌它反弹**。且 `ACUTE_PRICE_PULLBACK` 当前样本极小，不能把这类红旗当稳定正信号。→ 名单必须写明"博反弹候选，非安全推荐"，防止新手误读。

## 7. 回溯机制（执行后持续做）

### 7.1 数据成熟时间表（市场日历，客观）
- 你这批最新 US 推荐：**美东 6-8 周一收盘** → 系统 **中国时间 6-9 周二早** 回填 1D outcome。
- 5D：**6-10 之后**第一批。
- 20D：**7 月初**才有判断力。
- （NYSE 下一休市 6-19，6-8 是正常交易日。）

### 7.2 每轮回溯看的指标（跑同一套体检）
1. 严筛口径 1D：样本 / 胜率 / alpha，对比"全量 US buy"。
2. 分层是否仍成立：Top1-5 vs Top6-20；momentum<80 vs ≥80。
3. OVERHEATED_1Y 是否仍 0 胜率。
4. 严筛具体票逐只 alpha（看是不是个别票拖累）。

### 7.3 放行 / 退回决策树
```
6-9 早 1D 成熟 → 跑体检
├─ 上线后严筛前瞻样本 reviewed≥20 且 alpha>0 且 命中≥45% ？
│   ├─ 是 → 再看最近两轮是否都达标 → 连续两轮过 → 仅可提交"可小仓试探"人工评审
│   ├─ reviewed<20 → 继续作为买前研究队列观察，不推进真钱评审
│   └─ reviewed≥20 但 alpha/命中不达标 → 进失败归因（§7.4），不继续盲等样本
└─ 严筛口径 alpha 连续两轮转负，且每轮成熟样本≥3 → 降级为折叠研究观察；
   累计 reviewed≥20 且 alpha 仍为负 → 自动撤下严筛区，退回纯研究
```

**对称性原则**：退回要等 `reviewed≥20` 防误杀，放行也必须等 `reviewed≥20` 防误放。折叠虽然只是降级显示，也必须要求单轮成熟样本≥3，避免 1-2 个样本的噪声把整块研究区折叠。历史回测的 49 个样本只能支撑"研究队列"，不能单独支撑"可小仓试探"。

### 7.4 失败归因清单（alpha 仍不过时逐条查）
- [ ] **momentum 权重/方向**：是否该从 0.15 进一步降到趋近 0？（5-26 IC 审计 + 本次两次证据）
- [ ] **价格红旗松紧**：OVERHEATED_1Y 该不该更硬？其他红旗有没有误拦/漏拦？
- [ ] **反转因子**：reversal 是真 alpha 还是这波回调的巧合？
- [ ] **主题权重**：tech_ai 高 beta 是否在回调期需要降仓位？
- [ ] **推荐窗口**：1D 太短是否误判？要不要以 5D 为主口径？

## 8. 风险与缓解

| 风险 | 后果 | 缓解 |
|---|---|---|
| 小样本过拟合（momentum<80） | 市场风格切换后反向亏 | 标🧪 + 回溯单独盯 + 自动退回 |
| 用户误读严筛=可买 | 拿博反弹候选当真钱买 | 文案"博反弹·仅研究"+ 不自动买 |
| "Top1-5"口径漂移 | 与对方原分析对不上 | D1 已钉死；实现必须输出 `market_rank` 和 `global_rank` 供验收 |
| 风格逆风被当策略失效 | 过早推翻系统 | 保留"七成逆风/三成真坏"判断，别一周亏就改公式 |
