# B 路线 — 个股深度研究系统

> 维护: yanli (lance7in@gmail.com) · 创建: 2026-05-10 · **不构成投资建议**

## 起源

v8 系统在 **量化打分**（横截面 IC + 因子 + 组合优化）做到 97/100，但用户问"系统能做到仔细专业的财报分析么"时，暴露了根本缺失：

> 系统是 **buy-side 量化打分**，不是 **sell-side 个股纵深**。专业机构这是两个工种，本系统只做了一半。

B 路线就是把另一半补上。目标对标萝卜投研 / 摩根士丹利研报水平的中文机构级深度报告，但保持本系统已有的"量化骨架 + 多源审计"差异化优势。

## 关键技术决策

### 1. 不引入新依赖

`claude_client.py` 用 `requests` 而非 `anthropic` SDK；`sec_filings.py` 用正则而非 BeautifulSoup。原则：项目已 6 个开源库依赖，能不加就不加。

### 2. SEC 章节切分用反向贪心

10-K 目录里每个 Item 都密集出现，正文里 Item 按编号严格递增。**反向贪心**（从 Item 15 倒着选 < 已选位置的最大匹配）自动跳过目录条目，比"最后一次匹配"或"前向贪心 + skip head" 都稳健。NVDA 实测 19 个章节切得 100% 正确。

### 3. Beneish 在高增长股的假阳性 caveat

NVDA 营收年增 65% 时 SGI = 1.65，单这一项推高 M-Score +0.59，导致误标 HIGH RISK。代码自动识别 `SGI > 1.5`，标记为 `growth-driven false-positive` 而非 manipulation risk。出处：Beneish & Nichols (2007)。

### 4. Altman X4 cap

零负债公司（如 NVDA）X4 = 市值/总负债 会爆炸（实测 105），按 Altman 自己 2000 年论文建议 cap = 10。

### 5. Prompt caching 设计

`claude_client.complete(cache_system=True)` 把 system prompt（含 10-K 全文 ~80K tokens）标 ephemeral cache。同股第二次跑命中 cache，成本从 $0.5 降到 $0.05（94% 折扣）。

## 触发原因 — Phase 1 的 3 个硬伤

| # | 问题 | v9.1 解 |
|---|---|---|
| ❶ | 用 Trailing P/E 43 判定 NVDA "贵",**对成长股是反向信号**(市场按 forward 25-28 定价) | [forward_valuation.py](../stock_research/core/forward_valuation.py) |

| ❻ | 静态指标(cur vs prev)**丢失方向信息** — ROE 101% 是从 60% 涨上来还是 130% 跌下来? | [quarterly_trends.py](../stock_research/core/quarterly_trends.py) |

## 新增 3 个 core 模块(共 778 行)

### `forward_valuation.py` — Forward 估值倍数

- FY1/FY2/FY3 forward P/E + forward EV/Sales(基于 FMP 分析师一致预期)
- PEG (Lynch 1989 标准): forward P/E / 未来 EPS 2 年 CAGR,带阈值带(< 1.0 / 1.0-1.5 / 1.5-2.0 / ≥ 2.0)
- 诚实标注:"forward EBITDA 免费层不可得,用 EV/Sales 替代"
- CLI: `python3 -m stock_research.core.forward_valuation NVDA`

### `dcf_scenarios.py` — 自建 DCF 三档场景

- 显式 5 年 + 终值 Gordon model 标准公式
- 三档假设全部显式可审计:
  - 🐻 保守: 增速 -2pp / 终值 FCF margin 0.7× / WACC 11% / TGR 2%
  - 🐂 基准: 增速 ±0pp / 终值 FCF margin 0.85× / WACC 9% / TGR 2.5%
  - 🚀 乐观: 增速 +2pp / 终值 FCF margin 1.0× / WACC 8% / TGR 3%
- 5×5 敏感度矩阵(WACC ∈ [7%, 11%] × TGR ∈ [1.5%, 3.5%]),每格自动绿/红/黄染色 vs 现价
- 终值占比披露(过高警示估值脆弱)
- CLI: `python3 -m stock_research.core.dcf_scenarios NVDA`

### `quarterly_trends.py` — 8 季财务 trend

- 10 个指标 8 季时序(毛利/营业/净利率 + YoY/QoQ + DSO/DIO + Sloan TATA + CFO/NI + SBC%)
- 自动 trend 判定: improving / deteriorating / stable / insufficient_data
- 反向指标(DSO/DIO/SBC% 等"越低越好")**语义自动翻转**,不会把"周转天数下降"误标 deteriorating
- 趋势算法: 序列前一半均值 vs 后一半均值,相对变化 > 5% 才判改善/恶化(避免噪声)
- CLI: `python3 -m stock_research.core.quarterly_trends NVDA -n 8`

## 新增 FMP 缓存层

### `adapters/fmp_cache.py` — 24h 文件缓存

**为什么必须做**:加完 v9.1 三模块后,单份基本面研报调用 FMP 30+ 次(年报 5 次 + 季报 24 次 + peers loop 6-8 次),免费档 250 calls/day **跑 7-8 只就爆**。

**实现**:
- 文件缓存,**零依赖**(不引入 SQLite/Redis)
- 路径: `data/cache/fmp/<sha256_24>.json`
- key = hash(path + sorted_params),params 顺序无关,排除 apikey
- 24h TTL 自动失效(财报日次日缓存自然作废)
- 透明接入 [fmp_client._get](../stock_research/core/fmp_client.py),**调用方零侵入**
- **只缓存成功响应** — 429 / null / Error Message 不缓存(下次重试)

### 缓存 6 项单测全部通过

| # | 测试项 | ✅/❌ |
|---|---|---|
| 1 | save/get 一致性 | ✅ |
| 2 | None 不缓存(429 透明传递) | ✅ |
