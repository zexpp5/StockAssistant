# 2026-05-10 A 股实战层补全 — 10 项功能落地

## 起源

诊断发现：现有 v8 系统是**美股机构研究系统**（评分 97/100），但 A 股零售
实战缺口大 —— Markowitz 算出的 12% 仓位实盘可能：
- 标的在涨停板买不进去（成交 = 0）
- 标的被 ST 后涨跌停从 ±10% 变 ±5%，风险特征完全变了
- 标的次日停牌完全无法出场
- 标的 7 日内有大额解禁但模型没看见

按"阻止亏钱 → 提升信号 → 用户高频需求 → 进阶"四个优先级补 10 项。

## 关键技术决策

### 1. 用一次 spot_em 调用代替 N 次个股查询
A 股 5000+ 只股票，`fetch_spot_snapshot()` 一次拉全量（200KB JSON），本地派生
ST/涨跌停/停牌/板块 4 个状态，避免 N 次 API call 触发 akshare 限流。
缓存 5 分钟 TTL（盘中可承受的延迟）。

### 2. ST 主板的涨跌幅自动降为 ±5%
原规则：A 股主板 ±10%；ST 主板 ±5%；科创/创业 ±20%。
[a_share_filters.py:115](../stock_research/core/a_share_filters.py#L115) 在
`_classify_board()` 之后判断 ST 状态，自动覆盖 `limit_pct` 字段。
后续 Markowitz 权重约束直接用此字段。

### 3. 流动性约束用 20 日均成交额而非 ADV
`cap_by_volume_cn()` 默认 max_amount_pct=3%（比美股 5% 更保守，因为 A 股 T+1
+ 分时撮合冲击成本更高），且额外要求 20 日均额 ≥ 5000 万，否则直接拒绝
（保持原仓位）。

### 4. PEAD 因子用真实公告日，事件窗口 60 日
[event_calendar.py:202](../stock_research/core/event_calendar.py#L202) 的
`pead_factor()` 替代 [factor_model_china.py](../factor_model_china.py) 里硬编码
的"季报周期"逻辑：
- `last_earnings_date(code)` 取真实公告日
- 在 [公告日, 公告日+60] 窗口内 + surprise > 0 才打高分
- 出处：Bernard & Thomas (1989) PEAD 60 日衰减实证

### 5. 政策事件评分 1-5 分综合权威性 + 主题数
[policy_events.py:142](../stock_research/core/policy_events.py#L142) 的 `_score()`：
- 国务院/中央 → 3 分；部委 → 2 分；其他 → 1 分
- 命中政策动词（"印发"、"通知"等）+ 1 分
- 命中至少 1 个主题 + 1 分；命中 ≥ 3 个主题再 + 1 分
- 总分 1-5；min_score=2 是默认过滤门槛

### 6. 北向"双重确认"信号
[north_flow_signals.py:185](../stock_research/core/north_flow_signals.py#L185) 的
`double_confirm_signal()`：北向连续 5 日加仓（>0.3pp）+ 因子分上升 >5% = 强买入
信号。理论：纯北向跟踪有滞后；纯因子模型缺事件视角；两者交集是最高质量信号。

### 7. Walk-forward 用 IC 加权而非完整 ML 重训
为减少复杂度，简化版 [walk_forward_backtest.py](../stock_research/jobs/walk_forward_backtest.py)
只校准"动量 + 反转"两因子的等权权重。后续可扩展为：
- 完整因子集 IC 加权（factor_ic.py 已有）
- LightGBM 月度重训（financial_ml.py 已有 Purged K-Fold 框架）

## 闭环集成（v9.0 第二轮交付）

第一轮的 8 个模块只是"信号 / 数据层"，本轮把它们串到现有决策流水线，形成真正的闭环：

### 新增 2 个集成 job

| 文件 | 作用 |
|---|---|
| [a_share_picks.py](../stock_research/jobs/a_share_picks.py) | A 股选股 6 因子闭环 → `data/a_share_picks.json` |
| [apply_a_share_constraints.py](../stock_research/jobs/apply_a_share_constraints.py) | 对 plan_a_v5 的 A 股仓位应用实战约束 → `plan_a_v5_constrained.json` |

### a_share_picks 闭环数据流

```
飞书 watchlist (A 股)
    │
    ├─→ Piotroski F-Score   (factor_model_china)        权重 0.25
    ├─→ 12-1 月动量          (factor_model_china)        权重 0.15
    ├─→ 1 月反转            (factor_model_china)        权重 0.10
    ├─→ 龙虎榜机构净买入     (lhb_signals)                权重 0.15
    ├─→ 北向资金信号        (north_flow_signals)         权重 0.15
    ├─→ PEAD 真实公告日     (event_calendar.pead_factor) 权重 0.10
    └─→ 政策主题加成        (policy_events.tailwind)     权重 0.10
                │
                ▼
        加权综合分 (横截面归一化)
                │
                ▼
    × event_calendar.risk_score (0-1)   ← 解禁/减持降权
                │
                ▼
    a_share_filters.filter_tradable()   ← ST/涨停/停牌剔除
                │
                ▼
        Top K → data/a_share_picks.json
```

### apply_a_share_constraints 后处理

```
plan_a_v5.json (Markowitz 输出)
    │
    ├─→ 美股仓位（不动）
    └─→ A 股仓位
        │
        ├─→ 约束 1: ST/涨停/停牌 → 权重清零
        ├─→ 约束 2: event_risk_score (0-1) → 权重×风险分
        └─→ 溢出权重 → 现金
        │
        ▼
    plan_a_v5_constrained.json
```

## 设计哲学回顾

本次实施严格遵守现有项目原则：
- **不引入新依赖**：所有 8 个模块只用已有的 akshare + pandas + dataclasses
- **纯函数 + dataclass**：所有信号模块返回 dataclass，无 I/O 副作用
- **横截面友好**：所有因子返回 0-1 分数，便于 daily_picks 横截面排序
- **学术可追溯**：每条规则附论文/出处（见各模块文件头）
- **CLI 自测**：每个模块都有 `python -m ...` 入口，便于独立验证
