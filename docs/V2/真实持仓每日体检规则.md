# 真实持仓每日体检规则

更新时间：2026-05-22

## 定位

真实持仓每日体检只评价 `real_holdings` 里的用户真实持仓。它不创建股票池，不写入 `recommendation_picks`，也不改 `model_sim_holdings`。

页面上有两种口径：

- 账户总览：实时估值，按页面当前能拿到的行情和汇率计算。
- 今日持仓体检：跑批快照，写入 `real_holding_review_runs` / `real_holding_review_items`，用于历史复盘“当时系统为什么这么判断”。

## 分层

- `coverage_class`：说明数据覆盖到哪一层。
  - `ai_portfolio`：AI 组合有目标仓位。
  - `picks_only`：普通股票评分，不进入 AI 组合目标。
  - `tracking_only`：ETF、黄金、债券等，只做风控跟踪。
  - `needs_fix`：代码或行情阻塞，先补数据。

- `treatment_class`：说明系统实际采用哪种分析方法。
  - `portfolio_model`：AI 组合目标 + 股票评分 + 风控判断。
  - `stock_score`：普通股票评分 + 风控判断。
  - `risk_only`：只看市值、盈亏、止损和集中度。
  - `data_blocked`：数据不足，不能下判断。

## 阈值来源

阈值来自 `user_config.real_holding_review_rules`，默认版本为 `v1_guardrail_2026_05_22`。

这组阈值是保守 guardrail，不是已回测验证的 alpha。后续要通过样本积累、消融测试和策略验证再校准。

默认值：

- `loss_review_pct = -12.0`：股票浮亏达到该水平，进入风险复查。
- `tracking_loss_review_pct = -8.0`：黄金/ETF 等跟踪资产浮亏达到该水平，进入风险复查。
- `stop_breach_score_cap = 30.0`：破止损线时评分封顶。
- `loss_score_cap = 45.0`：大幅浮亏时评分封顶。
- `watch_score_cap = 55.0`：接近止损或模型转弱时评分封顶。
- `near_event_score_penalty = 5.0`：临近事件时评分扣分。
- `missing_price_score_penalty = 15.0`：缺行情时评分扣分。
- `weak_score_threshold = 55.0`：低于该分数进入减仓观察。
- `add_watch_min_score = 70.0`：只有达到该分数才允许出现关注加仓。
- `underweight_add_gap_pt = -2.5`：低于 AI 目标仓位 2.5pt 且评分合格，才出现关注加仓。
- `score_snapshot_max_age_days = 3.0`：自选股评分缓存最多允许使用最近 3 天，避免周末/节假日刚过午夜就把持仓降成“无评分”。

覆盖度默认值：

- `coverage_base = 0.20`
- `coverage_price = 0.30`
- `coverage_model_score = 0.30`
- `coverage_target_or_tracking = 0.20`

## 建议规模（advisory · 2026-05-25）

`real_holding_review` 在 `_build_item` 中附加 `size_advisory`（写入 `size_advisory_json`）：

- **关注加仓**：按 AI 目标仓位差额 + 半 Kelly 上限（`max_single_pct × kelly_fraction`，默认 15%×0.5）估算可参考股数。
- **风险复查 / 减仓观察 / 超 25% 红线**：给出可参考减仓规模；文案固定「可考虑分 3 批，每批 ≤ 1/3」。
- **仅风控跟踪 / 补数据**：通常无规模建议。

阈值键：`kelly_fraction`、`max_single_pct`、`hard_single_cap_pct`、`suggested_batches`（见 `user_config.real_holding_review_rules`）。

看板持仓表列「建议规模」只渲染后端字段，前端不重算。

## 板块热度 badge（2026-05-25）

`industry_heat` 来自 `openbb_intel` 的 `sector_rotation`（或实时 `sector_etf`），按持仓 theme/industry 映射 GICS ETF，在股票名列显示 60d 涨跌幅与 hot/cold/neutral。

`defense_watcher` 另含持仓日内告警：单票 ≤-5% 或组合加权 ≤-1.5%（收盘 vs 前收）。

## 当前限制

- 没有当日股票评分时，只能显示“补数据”，不会假装已经完成推荐。
- 黄金/ETF 等资产不使用股票因子模型评分。
- `GET /api/real-holdings/daily-review/latest` 只读快照，不临时重算。

## 股票评分来源

真实持仓体检不另造一套选股模型。股票评分只从已有推荐/评分链路读取：

- AI 组合覆盖的股票：优先读取 `recommendation_picks` / `portfolio_plans` 的生产结果。
- 美股普通股票：优先读取 `factor_scores_today.json.watchlist_scores` 的最终评分；如果只有原始因子缓存，兜底重算时必须使用同一个 `factor_weights_used`，避免 IC gate 权重漂移。
- 港股普通股票：复用 `stock_research.core.hk_scoring`，与 `scripts/pipeline/hk_picks.py` 同一套 composite、coverage、sector cap 和评级阈值。

港股评级阈值与港股主路径一致：

- `composite >= 0.75`：⭐⭐⭐ 强烈推荐。
- `composite >= 0.60`：⭐⭐ 推荐。
- 其余为 ⭐ 关注。

真实持仓体检不会为港股单独发明 `⛔ 不建议` 档；如果港股主路径未来调整阈值，持仓页应通过 `hk_scoring` 自动同步。
