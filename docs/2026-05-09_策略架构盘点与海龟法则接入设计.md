# 策略架构盘点与海龟法则接入设计

> 日期：2026-05-09
> 主题：当前系统是单策略还是多策略？海龟法则接入后会怎样？多策略竞争如何展示？
> 维护者：yanli · **不构成投资建议**

---

## 一、当前系统的策略全景

### 1.1 一句话总结

**架构是"单入口选股 + 多层防御"，不是真正的多策略竞争。**

只有一条选股管线（v5 五因子），所有版本演进做的都是"在同一个入口上叠防御层 / 加审查器"，从来没有第二条独立产出 picks 的策略与之并行竞争。

### 1.2 模块分层

| 层 | 模块 | 角色 |
|---|---|---|
| **选股入口（仅一条）** | [daily_picks_v5.py](../daily_picks_v5.py) + [factor_model.py](../factor_model.py) | 5 因子学术模型：Piotroski F-Score + 12-1 月动量 + 1 月反转 + 营收 PEAD + 分析师上调 |
| 旧版备份（已退役） | [daily_picks.py](../daily_picks.py) | 4 维启发式打分（35/25/25/15），被 v5 替代 |
| **组合优化** | [stock_research/jobs/optimize_portfolio.py](../stock_research/jobs/optimize_portfolio.py) | 因子中性化 + Markowitz + ADV 限流 + 成本扣减 |
| **防御层（叠加，非竞争）** | [trend_filter.py](../trend_filter.py)、[regime_filter.py](../stock_research/core/regime_filter.py)、[defense_signals.py](../stock_research/core/defense_signals.py) | Faber 200MA → VIX>30 → -15% 个股止损（v7 实盘版） |
| 早期信号（确认型） | [early_signals.py](../early_signals.py) | 内部人买入 + 分析师上调 |
| **审查 / 对比** | [stock_research/jobs/stress_test.py](../stock_research/jobs/stress_test.py) | A/B/C 三路对比：原版 / +200MA / +200MA+VIX+止损 |
| 业绩归因 | [pyfolio_tearsheet.py](../stock_research/jobs/pyfolio_tearsheet.py)、[factor_tearsheet.py](../stock_research/jobs/factor_tearsheet.py) | Pyfolio 全套 + 因子级 IC 时间序列 |

### 1.3 关键事实

- **stress_test.py 已经实现了"防御变体对比"框架**，但对比的是同一选股池上的不同防御开关，不是不同选股逻辑。
- **系统完全没有 turtle / Donchian / ATR / N 日新高**等突破型逻辑（grep 全库无命中）。
- 这是因子派系统，与海龟（趋势跟随派）属于两个流派，互补而非冲突。

## 二、海龟法则接入：理论预期

### 2.1 海龟原版核心

> 20 日新高入场 + 10 日新低出场 + ATR 头寸 + 2N 止损

它的设计目标和现有系统**根本不同**：

| 维度 | 现有 5 因子 | 海龟 |
|---|---|---|
| 信号源 | 财报 + 价格历史（季频更新） | 纯价格突破（日频） |
| 持仓周期 | 月度~季度 | 几天~几个月（趋势期长） |
| 选股池 | watchlist 37 只精选 | 适合流动性好的大池子（原版用期货组合） |
| 强项市场 | 横盘 + 成长（因子分散） | 单边趋势（牛市 / 崩盘） |
| 弱项 | 趋势末端 + 急跌（已用 v7 防御补） | 震荡市连续假突破（"鞭打" / chop） |

### 2.2 接入后的合理画像（理论预期）

| 市场场景 | 海龟相对 5 因子的表现 | 说明 |
|---|---|---|
| 2020 年 4-12 月、2023 年 AI 行情 | 海龟更快上仓位，alpha 显著 | 单边趋势是海龟主场 |
| 2022 加息震荡 | 海龟被 chop 掉，5 因子 + v7 防御反而更稳 | 震荡市海龟弱点 |
| 崩盘期（2008/2020/2018） | 10 日新低出场比 200MA 更敏感，回撤理论上比 v7 C 终极版（-13%）更小，但反弹也更慢 | 灵敏度 vs 假信号的平衡 |
| **与现有信号的相关性** | 12-1 动量因子和海龟 20 日突破有部分重合 | **必须 IC 实证**，增量可能不如想象 |

### 2.3 三种接入定位（择一推进）

#### 方案 1：作为第 6 个因子（最轻量）
- 把"距 20 日新高的距离"标准化后并入 [factor_model.py](../factor_model.py) 的 z-score 合成
- 让它和动量、F-score 一起打分
- 成本最低，能立刻知道海龟信号在 watchlist 上是否有增量

#### 方案 2：作为独立第二选股策略（中量）
- 新建 `daily_picks_turtle.py`，产出独立 picks 列表
- 让它和 v5 picks **真正竞争**——这才是"多策略"
- 触发条件：方案 1 的 IC > 0.05 且与 v5 相关性 < 0.5

#### 方案 3：作为防御层信号（最贴合现有架构）
- 用 10 日新低作为比 -15% 止损更早的退出信号
- 并入 [defense_signals.py](../stock_research/core/defense_signals.py)
- 不改选股，只改退出

**推荐顺序**：方案 1 → IC 验证 → 方案 2 升级，方案 3 可平行做。

## 三、多策略竞争对比的展示设计

[stress_test.py](../stock_research/jobs/stress_test.py) 的 A/B/C 表格是好底子，但它对比的是**同一策略的不同防御**。要做"策略竞争"，需要扩成两个维度。

### 3.1 主表：策略 × 指标

```
                  CAGR    Sharpe   MaxDD   Calmar   Turnover  Hit%   |  IC vs v5
──────────────────────────────────────────────────────────────────────────────
v5 五因子          18.2%    1.31   -32%    0.57       0.8      54%   |   1.00
v5 + v7 防御 (C)   16.5%    1.48   -13%    1.27       1.1      54%   |   0.94
海龟 20/10         14.0%    0.92   -22%    0.64       3.2      38%   |   0.31  ← 增量价值在这
海龟 + 因子门      17.8%    1.45   -15%    1.19       2.0      48%   |   0.62  ← 组合策略
```

**关键栏：IC 相关性**（与 v5 的相关度）
- 低相关 = 真分散，组合后能降低整体波动
- 高相关 = 重复信号，没有接入价值

### 3.2 三个辅助面板

1. **横截面对比**：4 次历史崩盘 × 4 个策略，复用 stress_test 的 4 行表（2008 Lehman / 2020 COVID / 2022 加息 / 2018 贸易战）
2. **Regime 切片**：按市场状态分桶展示（牛市单边 / 震荡 / 崩盘 / 复苏），暴露每个策略的"舒适区"
3. **持仓重叠热图**：当天 v5 picks 和 turtle picks 的交集 / 差集——最直观看出是不是真的两套独立判断

### 3.3 实现路径（顺现有架构）

- 在 [stock_research/jobs/](../stock_research/jobs/) 新建 `strategy_compare.py`
- 复用 [stock_research/core/factor_ic.py](../stock_research/core/factor_ic.py) 算两两 IC
- 输出沿用现有 HTML 仪表盘风格，加一个"策略竞技场"页签
- **守住已立的规矩**：权重 / 打分必须 IC 实证（见 [feedback_evidence_based_scoring.md](file:///Users/yanli/.claude/projects/-Users-yanli-Desktop----linearv-marketing/memory/feedback_evidence_based_scoring.md)），海龟参数也得回测，不能拍脑袋

## 四、推进路线图

| 步骤 | 动作 | 产出 | 决策点 |
|---|---|---|---|
| 1 | 实现方案 1（海龟作为第 6 因子）+ 单独跑 IC | 海龟信号在 watchlist 上的 IC、与 v5 相关性 | IC > 0.05 且相关性 < 0.5 → 进入步骤 2 |
| 2 | 升级方案 2（独立 picks）+ 搭建 strategy_compare.py | 真竞争对比、主表、3 个辅助面板 | 海龟独立策略 Sharpe > 0.8 且崩盘 MaxDD 优于 v5 → 入实盘候选 |
| 3 | 平行实现方案 3（10 日新低退出信号） | 接入 defense_signals.py，作为 v7 防御补强 | 减少 -15% 止损的滞后 |
| 4 | HTML 仪表盘加"策略竞技场"页签 | 多策略可视化对比 | 与现有股票仪表盘统一风格 |

## 五、待办与开放问题

- [ ] 海龟参数（20/10 vs 55/20）选哪个？原版 System 1 是 20/10，System 2 是 55/20，需在 watchlist 上各跑一遍 IC
- [ ] ATR 头寸是否要适配？watchlist 是股票不是期货，N 值的合理倍数需要标定
- [ ] 是否引入 Pyramiding（金字塔加仓）？海龟原版有，但与 Markowitz 优化器冲突，需要决策
- [ ] 持仓上限：原版海龟单品种最多 4 单位，多市场 12 单位——映射到股票池需要重新设计
