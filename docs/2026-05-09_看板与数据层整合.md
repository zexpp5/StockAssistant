### 3.2 跨源审计机制（独家）

[core/audit.py](../stock_research/core/audit.py) 自动比对多源 → 输出 4 级可信度：
- 🟢 **HIGH**：≥3 源一致
- 🟡 **MEDIUM**：单源 / 偏差 ≤ 1%
- 🔴 **LOW**：仅二手聚合
- ⚠️ **CONFLICT**：源间偏差 > 5%

写飞书"数据可信度"字段，给同事看的时候自动分级。

4. 写到 `history_data.json`（约 830 KB，84 只）

看板 JS 改成读本地 JSON，按时间窗口在前端切片：
```js
const PERIOD_DAYS = { "1mo": 22, "3mo": 65, "6mo": 130, "1y": 252, "2y": 504 };
```

| 维度 | 之前（直连）| 现在（本地）| 影响 |

## 7. 方法学纪律（与用户对话沉淀）

### 7.1 不给买卖建议（绝对边界）
- ❌ 不给 buy/sell 建议
- ❌ 不给价格目标
- ❌ 不给买入时点
- ✅ 可以做"上涨空间分析"（产业逻辑 / 估值 / 催化剂）
- ✅ 可以做风险标记 + 主题分级 + 时间轴位置

### 7.2 双层视角看 AI（用户反馈强化）

用户反驳了"过去 1 年是 AI 大牛市"的简化措辞。正确分两层：

| | 视角 | 数据 |
|---|---|---|
| **股价层面** | 是大牛市 | NVDA +330% / 中际旭创 +870% / AMD +240%（1.5 年）|
| **技术层面** | 仍处早期 | AI 渗透率 <10% / 数据中心电力占全美 ~3-4% / Robotaxi <1% |

**结论**：股价跑在业绩前面（牛市特征），但业绩还在加速兑现（早期特征）。仍处"百倍股 5 条件"中"认知反转 + 真实订单兑现"刚开始的阶段。

### 7.3 数据可信度 4 类标注

每条飞书记录"信息构成"字段必填：
- 📊 **实时事实**（WebSearch 当日抓取）：列具体数字 + 来源
- 📚 **历史事实**（公司基本盘、行业格局，相对稳定）
- 💭 **我的推断**（基于产业逻辑，需用户判断）
- 🤖 **训练数据已知**（模型训练截止 2026-01，可能滞后）

推断与事实必须**视觉上分开**。

### 7.4 自主执行边界（与用户协作纪律）

**✅ 不问直接做（增益型）**：
- 修 bug / 接权威源 / 补缺数据 / 改字段更准 / 删假数据 / yfinance → SEC EDGAR / 加 disclaimer / 修 IDE 误报

**⚠️ 必须先问（不可逆型）**：

## 13. 学到的 7 个教训

1. **架构走错了不要硬扛**：把代码迁移到正经 git 仓库 + 分层目录值 1 天工作量
2. **secrets 永远走 env**：`douyin_to_feishu.py` 含硬编码 secret 是定时炸弹
3. **graceful degrade > 完美主义**：akshare 限流时跳过比"等到通了"好
4. **免费层试水再付费**：FMP 试用发现免费层不覆盖你 watchlist 一半小盘股
5. **CORS 是浏览器问题，后端预拉解决**：`history_data.json` 模式
6. **数据准确性 >> 时效性**：1 天延迟无所谓，数据拉空才是大问题
7. **同事的视角比自己的视角更重要**：升级建议页就是为这个做的
