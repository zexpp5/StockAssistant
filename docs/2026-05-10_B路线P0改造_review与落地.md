## P0 落地（本次提交）

### 1. peer_compare 改造 — 修问题 ❸ ❽

[stock_research/core/peer_compare.py](../stock_research/core/peer_compare.py)

- `min_peers=4` 守门：样本不足时 percentile 直接返回 `None`，不再假冒分位
- `_resolve_peers` 二级回退：industry → sector → FMP 默认（marketcap）
- `INVALID_RANGES` 剔除指标无意义值：负 P/E / 负 EBITDA / 负 P/B / 负流动比率
- `peer_scope` 输出口径标注，区分 `same_industry / sector_fallback / user_specified`

**实测**：1 家同业时新版**正确**返回 None 分位；4 家同业时（AMD/INTC/TSM/AAPL）
NVDA 毛利率 90 分位 / 营收增速 90 分位 — 这才是 NVDA 真实位置。

### 2. forward_valuation — 修问题 ❶ 之 forward 倍数

[stock_research/core/forward_valuation.py](../stock_research/core/forward_valuation.py)

- 基于 FMP `/analyst-estimates` 年度一致预期
- 输出 FY1 / FY2 / FY3 的 forward P/E + forward EV/Sales
- 计算 PEG (FY1) = forward P/E / 隐含 EPS CAGR，按 Lynch 1989 阈值给判定
- 不算 forward EV/EBITDA：FMP 免费层不给 forward EBITDA，与其编不如不给

**为什么重要**：NVDA TTM P/E 43.6 看着贵 → 25 分位 → 误导；forward FY1/FY2 P/E
应在 25-28 区间，市场不是按 trailing 定价。

### 3. quarterly_trends — 修问题 ❻

[stock_research/core/quarterly_trends.py](../stock_research/core/quarterly_trends.py)

- `fmp_client.fetch_*` 三表函数加 `period='quarter'` 参数
- 拉 8 季三表对齐到共同日期序列
- 10 个时序指标：毛利率% · 营业利润率% · 净利率% · 营收 YoY% · 营收 QoQ% ·
  DSO 应收周转天数 · DIO 存货周转天数 · 应计盈余/资产% (Sloan TATA) · CFO/NI · SBC/营收%
- `_trend_direction`：序列前一半均值 vs 后一半均值，相对变化 >5% 标改善/恶化
- 反向指标（DSO/DIO/Sloan/SBC）方向语义自动翻转

### 4. dcf_scenarios — 修问题 ❷

[stock_research/core/dcf_scenarios.py](../stock_research/core/dcf_scenarios.py)

- 自建透明 DCF：所有假设全部参数化暴露
- Phase 1 显式 5 年：分析师一致 Revenue 预期 + growth offset → FCF 路径
- Phase 2 终值：Gordon growth `FV = FCF × (1+TGR) / (WACC - TGR)`
- **三档场景**：
  | | 增速 offset | 终值 FCF margin | WACC | TGR |
  |---|---|---|---|---|
  | 🐻 保守 | -2pp | 当前 × 0.7 | 11% | 2% |
  | 🐂 基准 |  0pp | 当前 × 0.85 | 9% | 2.5% |
  | 🚀 乐观 | +2pp | 当前 × 1.0 | 8% | 3% |
- **5×5 敏感度矩阵**：base case 下，WACC ∈ [7%, 11%] × TGR ∈ [1.5%, 3.5%] = 25 格 fair value
- 每格按 vs 现价 ±10% 自动染色（🟢/🟡/🔴）
